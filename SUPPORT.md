# Support

## Where To Ask Questions

- Use GitHub Issues for protocol questions, implementation bugs, and feature proposals.
- Use pull requests for concrete code changes.
- Use private security reporting for vulnerabilities.

## What To Include

- protocol version or commit SHA
- exact request and response payloads when relevant
- whether you used in-memory repositories or Postgres
- whether the flow was Lightning or on-chain
- any relay assumptions involved in the issue

## Response Expectations

This is an early-stage reference implementation. Best-effort support is provided, but production SLAs do not exist.

## Not Supported

- fiat or custodial payment logic
- ACP compatibility requests that require code reuse
- requests to put PII into relay-published events

# Code of Conduct

## Our Standard

This project expects respectful, technically rigorous collaboration.

Examples of behavior that contribute to a productive environment:

- discussing protocol and implementation tradeoffs in concrete terms
- giving actionable, evidence-based review feedback
- treating contributors with respect across experience levels
- focusing criticism on code, design, and behavior rather than people

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- discriminatory language or exclusionary behavior
- publishing private information without consent
- derailing technical discussion through hostility or bad-faith argument

## Enforcement

Project maintainers may remove comments, reject contributions, or restrict participation when behavior violates this code of conduct.

## Reporting

For conduct issues, contact the maintainers privately instead of escalating in public issue threads.

## Scope

This policy applies to repository discussions, issues, pull requests, and other project-managed communication channels.
